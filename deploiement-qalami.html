1 Préparer le VPS (une seule fois)
Bash — SSH en root sur le VPS 
# Créer un utilisateur dédié
adduser qalami
usermod -aG sudo qalami

# Copier ta clé SSH (depuis ta machine locale)
ssh-copy-id -i ~/.ssh/id_rsa.pub qalami@<SERVER_IP>

# Pare-feu UFW
ufw default deny incoming
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow <PORT_SSH>/tcp    # ex : 2222 (changer le port SSH par défaut)
ufw enable

# Installer Docker
apt update && apt install -y docker.io docker-compose-plugin
usermod -aG docker qalami
2 Créer les fichiers de config
docker-compose.yml — /opt/qalami/ 
services:
  frontend:
    build: .
    restart: unless-stopped
    env_file: .env
    networks: [internal]

  caddy:
    image: caddy:2-alpine
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data
    networks: [internal]

networks:
  internal:

volumes:
  caddy_data:
Caddyfile — /opt/qalami/ 
qalami.mr {
    reverse_proxy frontend:3000
    header {
        Strict-Transport-Security "max-age=31536000; includeSubDomains"
        X-Frame-Options "DENY"
        X-Content-Type-Options "nosniff"
    }
}
Dockerfile — racine du projet Next.js 
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public
EXPOSE 3000
CMD ["node", "server.js"]
next.config.ts — ajouter cette ligne 
output: 'standalone'
3 Configurer Cloudflare
  • Ajouter le domaine dans Cloudflare → pointer les nameservers chez ton registrar
  • DNS : enregistrement A — qalami.mr → <SERVER_IP> — proxy orange activé
  • SSL/TLS → mode Full (strict)
  • WAF → Rate limiting : 100 req/min par IP
  • Cache → règle bypass sur /* (Next.js SSR dynamique)
4 Fichier .env sur le serveur
/opt/qalami/.env — à créer manuellement sur le serveur 
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
SUPABASE_SERVICE_ROLE_KEY=eyJ...
NEXTAUTH_SECRET=<chaine-aleatoire-64-caracteres>
NEXTAUTH_URL=https://qalami.mr
⚠️ Ne jamais committer ce fichier dans git. Ajouter .env dans .gitignore. Ce fichier reste uniquement sur le serveur.
5 Premier déploiement
Depuis ta machine locale 
# 1. Vérification TypeScript (obligatoire avant tout push)
npx tsc --noEmit

# 2. Pousser le code
git push origin main
Sur le serveur (SSH) 
ssh qalami@<SERVER_IP>
cd /opt/qalami
git pull origin main
docker compose build frontend
docker compose up -d
docker compose ps    # tous les services doivent afficher "running"
6 Déploiements suivants (playbook)
Commande complète depuis la machine locale 
# 1. Gate local
npx tsc --noEmit

# 2. Push
git push origin main

# 3. Build + redémarrage sur le serveur (sans downtime)
ssh qalami@<SERVER_IP> '
  cd /opt/qalami &&
  git pull origin main &&
  docker compose build frontend &&
  docker compose up -d frontend
'

# 4. Vérifier
curl -s https://qalami.mr | grep -c "html"
