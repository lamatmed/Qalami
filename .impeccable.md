# Qalami — Design Context

> Référence persistante pour toutes les décisions de design UI/UX du projet.

---

## Design Context

### Users

**Interface admin école (priorité actuelle)**
- Directeurs d'école et secrétaires administratifs mauritaniens
- Utilisateurs desktop principalement ; certains utilisateurs âgés avec confort tech limité
- Connexion internet potentiellement lente ou instable (Mauritanie)
- Contexte bilingue : français comme langue principale, arabe (RTL) comme seconde langue

**Autres rôles (phases futures)**
- Enseignants : desktop + tablette
- Parents et élèves : mobile/tablette prioritaire

**Job principal** : gérer l'école au quotidien — inscrire des élèves, suivre les notes, organiser les classes et emplois du temps — sans friction technique.

---

### Brand Personality

**3 mots** : Fiable · Moderne · Humain

**Ton émotionnel** : L'interface doit inspirer confiance sans intimidation. Elle doit paraître professionnelle sans être froide ou trop institutionnelle. L'utilisateur doit se sentir compétent, pas perdu.

**Anti-référence** : Éviter le look "portail administratif d'État" — trop dense, trop gris, trop rigide. Pas de tableaux interminables sans respiration. Pas d'UI générique/bootstrapée.

---

### Aesthetic Direction

**Thème** : Light mode uniquement pour l'instant. Le dark mode existe dans le code mais n'est pas activé — ne pas le traiter comme une contrainte de design active.

**Palette par rôle** (validée) :
- Admin → Émeraude (vert)
- Enseignant → Indigo
- Élève → Ciel (sky blue)
- Parent → Ambre

**Typographie** :
- Corps : Inter (neutre, lisible, universel)
- Titres : Outfit (moderne, chaleureux)
- Arabe : Noto Sans Arabic

**Direction visuelle** :
- Espacement généreux — respiration visible entre sections
- Composants arrondis (border-radius moyen)
- Hiérarchie claire : titres → sous-titres → données
- Feedback d'état visible (chargement, succès, erreur) — connexion lente oblige

---

### Design Principles

1. **Clarté avant tout** — chaque écran doit avoir un seul focus principal. L'admin doit savoir d'un coup d'œil où il en est et quoi faire ensuite.

2. **Tolérance à la lenteur** — tous les états de chargement doivent être visibles et explicites. Les actions doivent donner un retour immédiat même si la réponse tarde.

3. **Accessible aux non-techniciens** — libellés clairs en français, pas de jargon. Les icônes sont toujours accompagnées de texte ou d'un titre tooltip. Contrastes WCAG AA minimum.

4. **Professionnel sans rigidité** — l'interface est sobre et efficace, mais avec des touches d'humanité (couleurs douces, arrondis, micro-feedback positif).

5. **Bilinguisme non-intrusif** — le français est la langue d'interface principale. L'arabe apparaît en complément (noms des niveaux, matières) avec direction RTL respectée localement, sans casser le flux LTR global.
